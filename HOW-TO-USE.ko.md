## 핵심

다음 턴을 보내기 전에 딱 이 질문에 답할 수 있어요:

> "지금 보내면, 캐시 혜택을 아직 받고 있나?"

---

## 먼저 볼 것

### 1. 상태 바 카운트다운

```text
TTL 42:15 · my-workspace
```

캐시가 만료되기까지 남은 시간이에요. 프로젝트 이름도 같이 보여서 창이 여러 개여도 구분돼요.

### 2. 툴팁

상태 바에 마우스를 올리면 직전 턴의 캐시 상태를 볼 수 있어요:

- 전체 입력 토큰 수
- 캐시 히트율 — 높을수록 좋아요
- 실제로 새로 처리된 토큰 수 — 낮을수록 절약된 거예요
- 캐시 상태 (cold start가 있었는지)

---

## 숫자 읽는 법

**Cache hit ratio가 핵심이에요.** 높으면 캐시 재사용, 낮으면 새로 처리된 거예요.

| 지표 | 뜻 | 좋은 방향 |
|---|---|---|
| Cache hit | 캐시에서 재사용된 비율 | 높을수록 좋아요 |
| Fresh input | 새로 처리된 토큰 | 낮을수록 절약 |
| Cache creation | 이번 턴에 새로 캐시에 쌓인 토큰 | 첫 턴이면 자연스러운 거예요 |
| Gross input | 모델이 본 전체 입력 크기 | 참고용 |

## 언제 뭘 쓸까

### `5분` — 빠르게 주고받을 때

- 버그 잡으면서 짧은 프롬프트를 연달아 보낼 때
- 1~2분 안에 다음 턴을 보내는 리듬일 때

### `1시간` — 천천히 생각할 때

- 코드나 문서를 오래 읽고 검토한 뒤 답할 때
- 설계, 기획, 리뷰처럼 턴 사이에 긴 사고가 필요할 때

한 줄로: 빠르면 `5분`, 느리면 `1시간`.

---

## 실전 시나리오

### 빠른 코딩 루프

1분 안팎으로 짧은 프롬프트를 계속 보내는 상황.

- `5분` 모드 유지
- TTL 남은 시간 체크
- cache hit이 높게 유지되면 리듬이 건강한 거예요

### 설계 / 리뷰 / 깊은 추론

문서를 한참 읽은 뒤에 다음 턴을 보내는 상황.

- `1시간` 모드가 안전해요
- 5분 모드였다면 읽는 사이에 캐시가 만료될 수 있어요

---

## 경고가 뜨면

### "TTL is under five minutes"

캐시가 곧 만료돼요. 보낼 게 있으면 지금, 한참 더 읽을 거면 `1시간`으로 전환.

### "Recent cache resets look frequent"

캐시가 자주 초기화되고 있어요. 작업 공백이 길다면 `1시간` 모드를 고려하세요.

---

## 프로젝트별 TTL 설정

### 기본: 전역 설정

상태 바를 클릭하면 `~/.claude/settings.json`이 바뀌어요. 모든 세션에 적용.

### 프로젝트마다 다르게

프로젝트 루트에 `.claude/settings.json`을 만들면 돼요:

**1시간 모드:**
```json
{
  "env": {
    "ENABLE_PROMPT_CACHING_1H": "1"
  }
}
```

**5분 모드:**
```json
{
  "env": {
    "FORCE_PROMPT_CACHING_5M": "1"
  }
}
```

프로젝트 설정이 전역보다 우선이에요.

## 롤링 상태 바

매 턴 완료 후, 상태 바가 잠깐 사용량을 보여주고 카운트다운으로 돌아가요:

```
[1] TTL 42:15          ← 기본: 카운트다운
[2] 84k 입력 · 히트 82%   ← 3초: 이번 턴 사용량 (배경색 변경)
[3] 5시간 25.6% (+2.1%) | 7일 42.0%  ← 3초: 누적 사용률 + 이번 턴 증가분
[4] TTL 42:09          ← 복귀: 카운트다운
```

- bridge가 연결 안 돼 있으면 3단계를 건너뛰어요
- 캐시 리셋 경고가 활성 상태면 롤링이 멈춰요

## 사용률 브릿지 설정

5h/7d 사용률 표시는 bridge가 필요해요.

### 작동 방식

1. Claude Code가 statusline을 통해 rate limit 정보를 출력
2. `bridge/write-rate-limits.js`가 읽어서 `~/.claude/ttl-counter-rate-limits.json`에 기록
3. 확장이 3초마다 그 파일을 읽음

### 수동 테스트

```bash
echo '{"rate_limits":{"five_hour":{"used_percentage":25.6},"seven_day":{"used_percentage":42.0}}}' | node bridge/write-rate-limits.js
```

이후 다음 턴 완료 시 사용률 flash가 표시돼요.

### 문제 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 5h/7d가 안 보임 | bridge 파일이 없음 | bridge 설정 또는 수동 테스트로 파일 생성 |
| 값이 바뀌지 않음 | bridge가 갱신을 안 함 | JSON 파일의 `updated_at` 확인 |
| delta가 (+0.0%)로만 뜸 | 이전 값과 현재 값이 같음 | 다음 턴에서 실제 변화 시 반영됨 |
